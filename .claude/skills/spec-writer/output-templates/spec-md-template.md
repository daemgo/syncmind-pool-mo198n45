# spec.md 文档模板

基于方案内容生成产品需求说明书时，使用以下模板结构。

**注意**：generate-spec-md Agent 的输出必须严格遵循此模板的章节结构和表格格式。

---

## 格式约定

- **区块标题**必须包含类型：`##### 筛选条件（form）`、`##### 数据列表（table）`
- **布局**用 backtick 标注：`` `list` ``
- **字段表**必须包含 fieldKey 列，使用 camelCase
- **选项来源**内联引用：`选项来源: dict-xxx`（dict-xxx 必须在全局规则 4.2 中定义）
- **路由**使用语义化英文路径：`/customers`、`/customers/:id`

---

```markdown

#### 约定说明

本文档使用以下标准值：
- **布局类型**: `list` / `detail` / `form` / `dashboard` / `steps` / `custom`
- **区块类型**: `table` / `form` / `card` / `cards` / `chart` / `tabs` / `steps` / `timeline` / `description` / `statistic` / `custom`
- **字段类型**: `text` / `textarea` / `number` / `money` / `date` / `select` / `multiselect` / `switch` / `upload` 等
```
