# 解决方案：会议纪要智能解析与销售跟进系统

## 1. 问题背景

销售团队日常通过钉钉闪击、线上会议与企业客户沟通，产生大量文字记录。目前这些信息散落在聊天记录中，无法系统化提取客户需求、销售进展和待办事项。团队需要从非结构化的会议内容中，快速生成结构化的客户洞察。

## 2. 核心能力

### 2.1 会议内容解析

- **输入方式**：粘贴钉钉闪击原文、会议记录文字
- **AI 自动解析**：
  - 提取客户名称（如能识别）
  - 提取客户明确的需求/痛点
  - 提取销售跟进进度描述
  - 提取所有 Action 项（待办、承诺事项）
  - 提取关键决策人/拍板人信息
  - 提取竞品对比信息（如有）
- **输出格式**：结构化解析结果，可一键存入销售跟进记录

### 2.2 统一汇总面板

- **多客户视图**：展示所有已解析客户的需求概览
- **按客户查看**：点击客户查看其完整需求列表、销售进度、待办 Action
- **Action 看板**：汇总所有待办事项，显示负责人和截止时间
- **进度追踪**：展示各客户的销售阶段（初访→需求确认→方案→商务→成交）

### 2.3 销售跟进记录

- 自动将解析结果写入 `followups.json` 格式
- 支持手动补充和修正解析内容
- Action 项支持勾选完成状态

## 3. 技术方案

- **前端框架**：React + TanStack Router（现有项目架构）
- **AI 能力**：通过 Claude API 进行会议内容解析，提取结构化信息
- **数据存储**：本地 JSON 文件（followups.json）
- **交互方式**：粘贴内容 → AI 解析 → 预览确认 → 写入记录 → 汇总展示

## 4. 页面结构

### 4.1 首页 / 汇总面板

- 客户列表（卡片形式，显示公司名、需求数量、跟进阶段）
- Action 汇总区（所有待办按截止时间排序）
- 快捷入口：粘贴新会议内容

### 4.2 解析页面

- 左侧：内容输入区（粘贴钉钉内容）
- 右侧：AI 解析结果预览
- 底部：确认写入 / 取消按钮

### 4.3 客户详情页

- 客户基本信息
- 需求清单（从历次会议解析提取）
- 跟进历史（时间线形式）
- 待办 Action 列表

## 5. 关键字段定义

| 字段 | 来源 | 说明 |
|------|------|------|
| companyName | AI 解析 / 用户确认 | 客户公司名称 |
| needs | AI 解析 | 提取的需求描述列表 |
| salesStage | AI 解析 + 用户选择 | 初访/需求确认/方案/商务/成交 |
| actions | AI 解析 | 待办事项：内容、负责人、截止日期 |
| decisionMaker | AI 解析 | 拍板人姓名和职位 |
| competitor | AI 解析 | 竞品提及（如有） |
| followupDate | 内容时间戳 / 当前日期 | 跟进日期 |
